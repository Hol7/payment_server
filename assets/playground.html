Gateway GraphQL Playground

This page provides ready-to-use GraphQL queries and mutations for testing the Gateway API at http://localhost:4000/api/graphiql

Note: Open GraphiQL IDE in another tab and copy-paste these examples. For payment mutations, set the Authorization header first.

1. Create a Merchant

mutation CreateMerchant {
  createMerchant(name: "Test Shop") {
    id
    name
    api_key
    inserted_at
  }
}

After running this: Copy the api_key from the response - you'll need it for payment mutations.

2. Set Authorization Header for Payments

In GraphiQL, click the "Headers" panel and add:

{
  "Authorization": "Bearer PASTE_MERCHANT_API_KEY_HERE"
}

3. Create a Payment

mutation CreatePayment {
  createPayment(amount: 1000, currency: "XOF", phoneNumber: "229XXXXXXXX") {
    id
    amount
    currency
    status
    phone_number
    internal_reference
    external_reference
    idempotency_key
    provider_id
    merchant_id
    inserted_at
  }
}

Expected: Payment created with status :initiated. Note the id for querying later.

4. Query Payment Status

query GetPayment($id: ID!) {
  payment(id: $id) {
    id
    amount
    currency
    status
    phone_number
    internal_reference
    external_reference
    provider_id
    merchant_id
    inserted_at
    updated_at
  }
}

Variables:

{
  "id": "1"
}

Status flow: :initiated → :processing → :success or :failed

5. List All Merchants

query ListMerchants {
  merchants {
    id
    name
    api_key
    inserted_at
  }
}

6. List All Payments

query ListPayments {
  payments {
    id
    amount
    currency
    status
    phone_number
    internal_reference
    external_reference
    provider_id
    merchant_id
    inserted_at
  }
}

Quick Test Workflow

  1. Create merchant → copy api_key
  2. Set Authorization header with the api_key
  3. Create payment → note id
  4. Query payment status to see updates

Troubleshooting

  • Payment creation fails: Check Authorization header is set with valid merchant API key
  • Payment stays :initiated: Check environment variables are loaded (MTN_BASE_URL, etc.)
  • No Oban jobs: In IEx: Repo.all(Oban.Job)
